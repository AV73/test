Thought process

This Temperature record application.I have created everything from scratch using Angular CLI

User Story #1 :

As end user you can be able to navigate to the "Records" page and see the "Add records" form.

  • Created a new page component Records and was linked to the nav component
  • The new page should include the jumbotron common component featuring the "Records for Temperaature" title.
  • This jumbotron is present a form with where the user can enter the values of temperature.
  • An Add button to add the temperature values are provided.
  • The new page component are unit tested for the presence of all elements described.

User Story #2 :

The input field is limited to maximum 8 temperature records

  • recordsCount is created that adds records in an array using arrow functions.
  • The input value should be stored upon the Add click using addRecords Form, the form should not allow more than 8 values
  • Disabled the add button after reaching 8 values using arrow disabled function

User Story #3

Created a progress bar to check the added records.

  • A process bar is created using ng-bootstrap
  • This reflect the number of records added / maximum records and should be filled accordingly.
  • The color of the progress bar should be green, when 8 maximum records are hit it should become red.
  • New unit tests are added to the page component spec file to cover the new functionality.

User Story #4

Validation is created using record.Valid.

  • Only numerical values are allowed by specifying type as number.
  • A pop up created for invalid input
  • New unit tests are added to the page component spec file to cover the new functionality.

User Story #5

Created Median

  • "Median" button is added next to the "Add" button.
  • Using ngbootstrap popover the median button gives the value
  • recordsCount is responsible for calculating the median.
  • Using disable option the median button is diabaled for no records
  • New unit tests are added to the page component spec file to cover the new functionality.
